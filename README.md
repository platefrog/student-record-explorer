# 학생부 탐색기 v1.0.1

학생부의 창의적 체험활동, 교과 세부능력 및 특기사항, 행동특성 및 종합의견을 전처리하고 특징어와 근거 문장을 탐색하는 교사용 로컬 분석 도구입니다. 커리어넷 공개 학과정보와 학생부 기록의 텍스트 연계도도 확인할 수 있습니다.

- 최신 배포판: <https://github.com/platefrog/student-record-explorer/releases/latest>
- 현재 버전: **v1.0.1**

이 프로그램은 학생의 진로·역량·합격 가능성을 판정하거나 학생을 평가·서열화하는 시스템이 아닙니다. 결과는 원문 검토와 상담 질문을 돕는 참고자료로만 사용해야 합니다.

## 실행

Windows에서 `run_dev.bat`를 실행합니다.

```powershell
.\run_dev.bat
```

터미널에 표시되는 `http://localhost:8501` 주소로 접속합니다. 종료할 때는 터미널에서 `Ctrl+C`를 누릅니다.

## 처음 사용하는 순서

1. `2. 학생부 전처리`에서 나이스 창체·교과세특·행발 `.xlsx` 파일을 올립니다.
2. 병합 미리보기에서 학년·반·번호·성명을 확인합니다.
3. Kiwi, MeCab 또는 간이 토큰화 중 현재 환경에서 사용할 수 있는 방식을 선택해 전처리를 실행합니다.
4. 전처리 결과를 ZIP, Excel, SQLite 중 하나로 저장합니다.
5. `3. 말뭉치 관리`에서 학과 말뭉치를 불러오거나 커리어넷 API로 갱신합니다.
6. 말뭉치를 검토하고 내장 DB로 저장합니다.
7. `1. 학생 분석`에서 학생과 분석 관점을 선택해 결과와 근거 원문을 확인합니다.

이후에는 원본 엑셀을 매번 전처리하지 않고 사이드바에서 전처리 결과를 불러와 사용할 수 있습니다.

## 나이스 원본 파일

- 창체: `나이스 > 학급담임 > 학교생활기록부 > 학생부 항목별 조회 > 창의적체험활동`
- 교과세특: `나이스 > 학급담임 > 학교생활기록부 > 학생부 항목별 조회 > 교과학습발달상황 > 세부능력및특기사항`
- 행발: `나이스 > 학급담임 > 학교생활기록부 > 학생부 항목별 조회 > 행동특성및종합의견`

학년·반은 엑셀 상단 또는 파일명에서 인식합니다. 권장 파일명은 `3학년 1반 창체.xlsx` 또는 `3-1반 창체.xlsx`와 같은 형식입니다.

## 주요 용어

- 전처리 캐시: 학생별 병합 원문, 빈도, TF-IDF, 근거 문장을 미리 계산해 저장한 결과입니다.
- Kiwi: 한국어 명사를 비교적 정교하게 추출하는 기본 형태소 분석기입니다. 최초 준비는 느리지만 같은 설정의 인덱스는 재사용합니다.
- MeCab: 빠른 형태소 분석기지만 한국어 패키지·사전과 Windows 실행 구성요소가 설치된 환경에서만 사용할 수 있습니다.
- 간이 토큰화: 글자 규칙과 간단한 어미 제거를 사용하는 빠른 방식입니다.
- TF-IDF: 전체 학생에게 흔한 단어보다 특정 학생 기록에서 상대적으로 두드러지는 단어에 더 큰 값을 주는 통계값입니다.
- 코사인 유사도: 두 문서의 단어 가중치 방향이 얼마나 비슷한지를 나타냅니다. 확률이나 합격 가능성이 아닙니다.
- 말뭉치: 학생부와 비교하는 학과 설명 텍스트 묶음입니다.
- 보완 키워드: 목표 학과 말뭉치에는 상대적으로 강하지만 학생부 기록에는 약한 표현입니다. 학생의 결핍을 의미하지 않습니다.
- 적합·높은 연계: 현재 불러온 학생 전체와 학과 전체의 텍스트 유사도 분포에서 각각 상위 1%, 상위 5%에 해당하는 표시입니다. 진로 적성이나 합격 가능성이 아닙니다.

## 학과 말뭉치

커리어넷 학과 상세정보를 다음 관점으로 나누어 분석합니다.

- 통합: 모든 학과 설명
- 학업: 학과 개요, 특성, 대학 주요 교과목
- 교과: 관련 고교 교과와 대학 교과목
- 활동: 진로 탐색 활동
- 적성: 흥미와 적성 설명
- 진로: 졸업 후 진출 분야, 관련 직업과 자격

커리어넷 갱신에는 승인된 API 키가 필요합니다. 키는 코드나 DB에 저장하지 않으며, 학생부 원문은 커리어넷으로 전송하지 않습니다. 수집 결과는 검토 후 `현재 말뭉치를 내장 DB로 저장`해야 다음 실행에서도 사용할 수 있습니다.

## 개인정보와 보관

- 로컬 실행에서는 학생부가 교사 PC에서 처리됩니다.
- 커리어넷 갱신 시에는 API 키와 공개 학과정보 요청만 외부로 전송됩니다.
- 업로드한 원본과 생성된 캐시에는 학생 식별정보와 학생부 원문이 포함될 수 있습니다.
- 임시 파일과 사용자가 저장한 결과 파일은 PC에 남을 수 있습니다.
- 외부 서버나 공용 PC에서 실행하면 로컬 처리 조건이 달라질 수 있습니다.

학교가 승인한 기기·계정·저장 위치를 사용하고, 접근 권한, 비식별화, 보관 기간, 결과 공유와 삭제 절차는 소속 기관 지침을 따라야 합니다.

## 결과 해석 시 주의

- 높은 유사도는 적성, 역량, 합격 가능성을 의미하지 않습니다.
- 낮은 유사도는 학생에게 경험이나 능력이 없다는 뜻이 아닙니다.
- 학생부는 학생의 모든 활동을 완전하게 담은 자료가 아닙니다.
- 단어 빈도는 활동의 중요성이나 학생의 우수성을 직접 나타내지 않습니다.
- 말뭉치, 불용어, 표현 통일 사전, 형태소 분석 방식이 바뀌면 결과도 바뀔 수 있습니다.
- 점수만 보지 말고 공통 키워드와 근거 원문을 함께 확인해야 합니다.

앱 안의 `5. 사용 안내` 탭에서 더 자세한 설명과 문제 해결 방법을 확인할 수 있습니다.
